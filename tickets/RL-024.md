# RL-024 — XGBoost Conditional Quantile Regression

## Status: Done

## Goal
Replace the broken XGB volatility model with a defensible ML use: conditional quantile regression for nonparametric VaR estimation. XGB now predicts VaR directly — no distributional assumption. Sits alongside the MC pipeline as a validation tool, not inside it.

## What changed

### New module
- **`src/analytics/xgb_var.py`**: `fit_quantile_model()` trains XGBoost with `reg:quantileerror` objective to predict conditional quantiles of next-day returns. `predict_var()` generates predictions from recent data. `backtest_quantile_var()` provides rolling walk-forward backtesting compatible with existing Kupiec/Christoffersen framework. Feature engineering reuses the same rolling features (vol, mean, skew, kurtosis, abs/sq return).

### Tests
- **`tests/test_xgb_var.py`**: 20 tests covering feature engineering, model fitting, prediction, backtesting (columns, breach consistency, step behavior), and quantile ordering (1% more extreme than 5%).

### Notebooks
- **`risk_analysis_walkthrough.ipynb`**: Replaced XGB vol section (8) with EVT (8) and XGB quantile regression (9). Updated intro table, imports, comparison table, and takeaways. Three-way VaR comparison: Normal vs EVT vs XGB.
- **`model_deep_dive.ipynb`**: Removed XGB-5d/10d/21d from MC configs (13→8 configs). Replaced XGB feature importances section with EVT + XGB quantile comparison. Updated imports, model fitting, representative models, backtesting configs, and takeaways.

## Decisions
- `reg:quantileerror` (not `reg:quantile`) — matches xgboost 3.2.0 API
- XGB sits alongside MC pipeline, not inside it — it's a validation/comparison tool
- No CVaR from XGB quantile regression (only VaR) — would need a separate model per quantile beyond VaR
- Walk-forward backtesting: train on rolling window, predict 1-step-ahead, same framework as MC backtests
