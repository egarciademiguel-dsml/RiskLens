# RL-007 — Monte Carlo Simulation + Analysis Notebook

## Objective

Implement Monte Carlo simulation for forward-looking risk estimation, add upside/opportunity metrics, and create an end-to-end visualization notebook.

## Scope

**Included:**
- `src/analytics/monte_carlo.py` — simulation engine, VaR/CVaR, scenario buckets, prob_target
- `src/analytics/risk_metrics.py` — added sortino_ratio, gain_to_pain_ratio, best_worst_periods
- `notebooks/risk_analysis_demo.ipynb` — full pipeline visualization

**Excluded:**
- Streamlit integration (RL-008)
- Tests (RL-009)

## Functions

**monte_carlo.py:**

| Function | Output | Purpose |
|---|---|---|
| `simulate_paths` | DataFrame (days × sims) | GBM forward price paths |
| `compute_var` | float | Value at Risk as percent loss |
| `compute_cvar` | float | Expected Shortfall beyond VaR |
| `prob_target` | float | Probability of reaching a specific return target |
| `scenario_buckets` | dict | Distribution across 5 outcome categories |
| `simulation_summary` | dict | Full summary including prob_gain, prob_loss, scenarios |

**risk_metrics.py (additions):**

| Function | Output | Purpose |
|---|---|---|
| `sortino_ratio` | float | Risk-adjusted return penalizing only downside |
| `gain_to_pain_ratio` | float | Total returns / total absolute losses |
| `best_worst_periods` | dict | Best and worst rolling N-day returns |

## Notebook Sections

1. Data ingestion
2. Risk + performance metrics (volatility, drawdown, Sharpe, Sortino, gain-to-pain, best/worst periods)
3. Returns distribution histogram
4. Rolling volatility
5. Drawdown chart
6. Monte Carlo summary (win/lose probability first, then VaR/CVaR, then upside targets)
7. Scenario distribution bar chart
8. Simulated price paths with percentile bands
9. Final price distribution with VaR/CVaR markers

## Decisions

- **GBM model**: Interpretable, standard, appropriate for MVP.
- **Prob gain/loss as lead metric**: First thing a non-quant user wants to know — displayed before VaR/CVaR.
- **5 scenario buckets**: Severe loss, moderate loss, flat, moderate gain, strong gain — intuitive for any audience.
- **prob_target**: Lets users ask "what's my chance of making X%?" — key for decision-support.
- **Sortino over Sharpe alone**: Sharpe penalizes upside volatility, which misleads for assets with positive skew (e.g., BTC).
- **Gain-to-pain ratio**: Simple, intuitive — "how much did I gain for every unit of pain?"

## Acceptance Criteria

- All functions return correct types.
- Notebook runs end-to-end and produces all 8 visualizations.
- Win/lose probability is prominently displayed.
- Scenario bar chart shows all 5 buckets with labeled percentages.
