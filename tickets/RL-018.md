# RL-018 — Regime Model Comparison Notebook

## Objective

Notebook comparing HMM vs GMM regime detection across 1, 2, and 3 regimes. Study how regime granularity affects drift, volatility estimates, and downstream MC risk metrics.

## Scope

**`notebooks/regime_comparison.ipynb`:**

### Structure

1. **Data & setup** — load asset, compute returns, fit both models at n_regimes = 1, 2, 3
2. **Regime assignments** — side-by-side time series with regime coloring (HMM vs GMM, for each n_regimes)
3. **Per-regime parameters table** — mu, sigma, observation count per regime, for all 6 combos (2 models × 3 regime counts)
4. **Transition matrices** — HMM transition probs for n=2 and n=3; GMM equivalent via observed transitions
5. **Regime influence on MC** — run `simulate_paths()` with each of the 6 configs, compare:
   - VaR / CVaR (95%, 99%)
   - Final price distributions (KDE overlay)
   - Path fan plots
6. **Regime count study** — how do risk estimates change as you go 1 → 2 → 3 regimes? Is the jump from 1→2 significant? Does 3 add meaningful signal?
7. **Key takeaways** — which approach captures regime structure better, where they agree/disagree, practical recommendation
