# RL-016 — HMM Regime Detection

## Objective

Implement Hidden Markov Model (HMM) regime detection using `hmmlearn`. Fit on historical returns to extract regime-specific drift (mu) and volatility (sigma), then integrate as a volatility/drift source in `simulate_paths()`.

## Scope

**Regime engine (`src/analytics/regime_hmm.py`):**
- `fit_hmm(returns, n_regimes)` → fitted model + regime params (mu, sigma, transition matrix per regime)
- `predict_current_regime(model, recent_returns)` → current regime index
- `get_regime_params(model)` → dict with per-regime drift, vol, transition probs
- Support `n_regimes` = 1 (baseline, equivalent to current global mu/sigma), 2 (calm/crisis), 3 (bull/sideways/bear)

**MC integration:**
- New `volatility_model="hmm"` option in `simulate_paths()`
- Uses current regime's drift + vol (not just vol — drift changes too)
- Optionally allow regime transitions during simulation via transition matrix

**Streamlit:**
- UI toggle for HMM regime model
- Regime count selector (1, 2, 3)
- Display detected regimes on price chart

**Tests:**
- Regime params shape and sign checks
- n_regimes=1 output ≈ global mu/sigma
- Integration with simulate_paths produces valid paths
