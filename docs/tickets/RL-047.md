# RL-047 — XGB as a Real Auditor: SHAP + Disagreement Detection + Richer Features

## Status: Done

## Problem
The project claims "ML audits parametric, doesn't replace" but the actual audit was just side-by-side VaR numbers. No explanation of *why* the models disagree, no systematic flagging of *when* they disagree, and the feature set was basic (tutorial-grade rolling stats, no volume or asymmetry features).

## Solution

### `src/analytics/xgb_var.py` — upgrades

**Richer features in `engineer_features`:**
- `rv_5d`: 5-day realized variance (sum of squared returns) — direct measure of recent realized risk
- `downside_vol_21d`: 21-day downside volatility (std of negative returns only) — captures asymmetric risk
- `volume_ratio`: current volume / 21-day mean volume (optional, when volume series provided) — liquidity signal
- `volume_vol_21d`: 21-day coefficient of variation of volume (optional) — volume regime signal

`volume` parameter is optional for backward compatibility. When omitted, volume features are absent.

**SHAP explanation via `explain_var`:**
- Uses `shap.TreeExplainer` on the fitted XGBoost model
- Returns SHAP values for all samples, base value, feature names, and feature importance (mean |SHAP| per feature, sorted descending)
- Enables: feature importance bar charts, SHAP summary plots, per-prediction waterfall plots
- Lazy import of `shap` — only loaded when `explain_var` is called

### `src/analytics/audit.py` — new module

**`detect_disagreements(parametric_var, xgb_var, threshold_pct=0.20) → DataFrame`**
Flags dates where relative divergence between parametric and XGB VaR exceeds the threshold. Returns the divergent dates with abs/rel diff and direction flag (`xgb_deeper`). This IS the audit: systematic identification of where parametric assumptions break.

**`regime_conditional_breach_rates(backtest_results, regime_labels) → DataFrame`**
Splits backtest breach rates by HMM regime. Shows whether a model's failures concentrate in crisis or calm — the pattern differs by model and reveals each model's structural weakness.

**`comparative_var_table(returns, var_estimates) → DataFrame`**
Multi-model VaR comparison: for each date, shows actual return + every model's VaR + per-model breach flag. Foundation for heatmaps and cross-model analysis.

## Decisions
- **Lazy import of shap.** `import shap` is inside `explain_var`, not at module top. Avoids slowing down all XGB imports for users who don't need explanation. Streamlit Cloud pays the numba JIT cost only if SHAP is invoked.
- **Volume features optional.** `engineer_features(returns, volume=None)` — backward compatible. Volume isn't always available or meaningful (some tickers have sparse/synthetic volume). The model works without it; adding volume is an enrichment.
- **`audit.py` is a separate module, not inside `xgb_var.py`.** Audit logic is model-agnostic — it compares any two VaR series. Coupling it to XGB would be wrong.
- **Disagreement threshold default 20%.** A 20% relative divergence between VaR estimates is large enough to be structurally meaningful, not just noise. Tunable per use case.
