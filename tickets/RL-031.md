# RL-031 — Multi-Seed Stability Analysis

## Status: Done

## Goal
Quantify how much VaR/CVaR estimates move across random seeds for each tier × asset combination, separating Monte Carlo precision from model accuracy.

## Scope
- New reusable module `src/analytics/seed_robustness.py` with `run_multi_seed()` and `robustness_summary()`.
- Notebook section that calls the module (not a parallel implementation).
- Reuse the same module from the Streamlit app if/when stability is surfaced as a feature.
- Save numerical results to a standalone report.

## Files changed
- `src/analytics/seed_robustness.py` (new)
- `notebooks/model_deep_dive.ipynb` (section 4b: Seed Robustness)
- `reports/seed_robustness_results.md` (new)

## Acceptance criteria
- 10 seeds × 3 tiers × 3 assets executed at 252-day horizon, 10k simulations each.
- Coefficient of Variation reported per metric.
- Stability vs. accuracy distinction made explicit (low CV ≠ correct).

## Decisions
- 10 seeds at `range(0, 100, 10)` is enough for CV stability without making the notebook slow.
- Single shared module rather than notebook-only code so the app can reuse it later.
- Reframed conclusions: MS-GARCH+EVT has the lowest CV but that is "stability of bias, not accuracy" — only the backtest can judge accuracy.
