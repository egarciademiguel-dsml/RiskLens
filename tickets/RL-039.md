# RL-039 — MS-GARCH Fit Quality Transparency

## Status: Done

## Goal
Surface in the Streamlit app *which* parts of the MS-GARCH fit are real GARCH/EVT and *which* are degenerate fallbacks. Originally framed as "per-regime GARCH fallback transparency" before RL-034 — reframed here because the unified-GARCH refit eliminated per-regime GARCH fallbacks entirely. Two failure modes still exist and need to be visible.

## Scope
- **Add fit-quality flags to `fit_ms_garch` return dict:**
  - `garch_fit_failed: bool` — True iff the global GARCH(1,1) fit on the pooled series raised and we fell back to constant-vol degenerate params (α=β=0).
  - `gpd_fit_failed_regimes: list[int]` — regime indices where the GPD fit was skipped (n < 50) or raised, so the simulator falls back to Normal shocks for that regime.
- **Rewrite the MS-GARCH info box** in `app/streamlit_app.py` to:
  - Show **one** global GARCH parameter line (α, β, persistence) instead of duplicating persistence per regime — α, β are shared after RL-034, so showing them per regime implies independence that no longer exists.
  - Show per-regime: long-run volatility, GPD shape (or "Normal ⚠️" with reason if fallback).
  - Show a top-level warning if `garch_fit_failed` is True.
- Do not modify the simulation engine or any other consumer of the dict.

## Files changed
- `src/analytics/ms_garch.py` — add two flags to the return dict.
- `app/streamlit_app.py` — rewrite the MS-GARCH info box to use the global+per-regime split and surface the flags.

## Acceptance criteria
- `fit_ms_garch` return dict contains both new keys for every successful call.
- Streamlit MS-GARCH info box shows global α, β, persistence on one line and per-regime σ_lr + tail shape on a second line.
- Per-regime GPD fallbacks are visually flagged (⚠️) with the reason ("insufficient obs" or "fit failed").
- Global GARCH fit failure shows a top-level warning.
- All existing callers of `fit_ms_garch` still work without modification (new keys are additive).

## Decisions
- **Reframed from the original ticket** — pre-RL-034, this would have shown per-regime fit success/failure. Post-RL-034 there is no per-regime GARCH fit, so the relevant transparency is at the global level.
- **Persistence shown once, not duplicated.** Hiding the fact that α, β are shared across regimes would misrepresent the model after RL-034. Showing it once is more honest and the change is documented in `docs/decisions/ms_garch_unified.md`.
- **GPD fallback gets a ⚠️ marker** rather than being silently labeled "Normal" — the user needs to know whether they are looking at a model choice (the bulk is Normal by design) or a fit failure (we wanted GPD and didn't get it).
