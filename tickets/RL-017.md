# RL-017 — GMM Clustering + Classifier Regime Detection

## Objective

Implement a two-step regime detection approach: Gaussian Mixture Model (GMM) to discover regimes from engineered rolling features, then a classifier to predict the current regime. Extract per-regime drift and volatility for MC integration.

## Scope

**Regime engine (`src/analytics/regime_gmm.py`):**
- `engineer_regime_features(returns)` → DataFrame with rolling features (vol windows 5/10/21d, rolling mean return, rolling skew/kurtosis, volume change if available)
- `fit_gmm(features, n_regimes)` → fitted GMM + regime labels
- `train_regime_classifier(features, labels)` → trained classifier (Random Forest or similar) to predict regime from current features
- `predict_current_regime(classifier, current_features)` → regime index
- `get_regime_params(returns, labels)` → per-regime mu, sigma
- Support `n_regimes` = 1, 2, 3

**MC integration:**
- New `volatility_model="gmm"` option in `simulate_paths()`
- Same integration pattern as HMM: regime-specific drift + vol

**Streamlit:**
- UI toggle for GMM regime model
- Regime count selector (1, 2, 3)
- Display detected regimes on price chart

**Tests:**
- Feature engineering output shape
- GMM labels in expected range
- Classifier accuracy > random baseline
- Per-regime params are distinct
- MC integration produces valid paths
