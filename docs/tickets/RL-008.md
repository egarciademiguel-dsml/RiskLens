# RL-008 — Streamlit App (Full UI)

## Objective

Build the interactive Streamlit dashboard wiring all existing modules (data ingestion, risk metrics, Monte Carlo) into a user-facing interface.

## Scope

**Included:**
- `app/streamlit_app.py` — full rewrite from bootstrap to working dashboard

**Excluded:**
- No new analytics or business logic — strictly UI/integration
- No new dependencies
- Tests (RL-009)

## UI Structure

- **Sidebar**: ticker input, date range, Monte Carlo params (horizon, simulations, confidence), Analyze button
- **Win/Lose probability**: two large metric cards — first thing the user sees
- **Key metrics row**: volatility, max drawdown, Sharpe, Sortino, gain-to-pain, best/worst periods
- **Scenario distribution**: color-coded bar chart with labeled percentages
- **Charts (tabbed)**: Price & Drawdown, Returns Distribution, Rolling Volatility, Monte Carlo Paths, Final Price Distribution
- **Upside targets**: probability table for +10%, +25%, +50%, +100%
- **Full summary**: expandable section with price outlook and risk details

## Decisions

- **Computation on button click only**: avoids redundant yfinance API calls on every widget change.
- **Tabs for charts**: prevents a wall of plots — user explores what interests them.
- **Win/lose as lead metric**: anchors the entire page for non-technical users.
- **No logic in the app file**: every computation calls existing functions from `src/`. The app is pure integration and layout.
- **Error handling via st.error + st.stop**: user sees clear messages, never tracebacks.
- **matplotlib/seaborn over Plotly**: consistent with notebook, avoids adding interactive chart complexity for MVP.

## Acceptance Criteria

- App runs with `streamlit run app/streamlit_app.py`.
- Invalid ticker shows error message, does not crash.
- All 5 chart tabs render correctly.
- Scenario bar chart displays all 5 buckets.
- Upside targets table is populated.
- No business logic lives in `streamlit_app.py` — all imported from `src/`.
