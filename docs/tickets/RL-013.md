# RL-013 — Fat-Tailed Monte Carlo (Student-t Shocks)

## Objective

Replace normal (Gaussian) shocks in the Monte Carlo simulation with Student-t distributed shocks to better capture tail risk. Real financial returns exhibit excess kurtosis (fat tails), so the normal assumption underestimates extreme events — making VaR/CVaR too optimistic.

## Scope

**Included:**
- `simulate_paths()` accepts `distribution="t"` and optional `df_t` (degrees of freedom)
- Auto-fits Student-t via MLE (`scipy.stats.t.fit`) when `df_t` not provided
- Standardizes t-shocks to unit variance: `Z = T / sqrt(df / (df - 2))`
- New `fit_t_distribution()` helper returns df, tail description, observation count
- Streamlit sidebar radio toggle: Normal vs Student-t
- Info banner showing fitted df and tail interpretation
- MC paths chart title reflects chosen distribution
- 12 new tests (8 simulation + 4 fit helper)

**Excluded:**
- No GARCH or stochastic volatility (separate ticket)
- No portfolio/multi-asset support (separate ticket)
- No manual df slider in UI (auto-fit is the default)

## Decisions

- **Clamp df >= 2.1**: variance formula `df/(df-2)` is undefined for df <= 2. Clamping to 2.1 keeps the math valid while still allowing very heavy tails.
- **Standardize shocks to unit variance**: dividing by `sqrt(df/(df-2))` ensures drift and sigma retain their original meaning from the GBM framework. Without this, t-shocks would inflate volatility.
- **Use `random_state` for t-distribution seeding**: scipy's `t.rvs(random_state=seed)` instead of global `np.random.seed()` to avoid side effects. Normal path keeps existing seed behavior for backward compatibility.
- **No manual df slider**: auto-fitting from data is the correct default for an MVP. Users who understand df can extend later.
- **Backward compatible API**: default `distribution="normal"` means all existing code and tests work unchanged.
