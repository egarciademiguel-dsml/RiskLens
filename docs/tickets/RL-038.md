## Decisions
- **Split rather than promote in place.** `model_deep_dive` answers "is each tier doing what it claims?" (validation arc); `horizon_crossover` answers "given a horizon, which tier should I pick?" (recommendation arc). Mixing dilutes both. The deep dive is already 45 cells.
- **Single source of truth.** The horizon §8 in `model_deep_dive` is removed entirely (Option A). Duplicated analysis rots.
- **Recommendation table is the deliverable.** This becomes the input for RL-041 (model recommendation logic in app).
- **All 3 tiers swept**, not just the vol/tail decomposition triplet (Baseline / GARCH+Normal / GARCH+Student-t). The original §8 only decomposed effects; the new notebook produces actionable VaR-by-horizon comparisons for the production tiers.

## Findings (post-run)
| Asset | Crossover | Sequence |
|---|---|---|
| BTC-USD | 63d | Baseline ×4 → MS-GARCH+EVT ×3 |
| NVDA | 126d | Baseline ×5 → MS-GARCH+EVT ×2 |
| SPY | 5d (oscillating) | MS-GARCH+EVT → GARCH+t ×3 → Baseline → MS-GARCH+EVT ×2 |

The intuitive prior — *crypto crosses earliest because tails are heaviest* — is **wrong**. SPY crosses first (5d) despite having the thinnest tails, because its low absolute volatility makes tier estimates close enough that small effects flip the ranking. NVDA crosses *latest* (126d) despite being the heaviest single-stock tail, because its single-regime σ stays high enough that Baseline keeps winning until regime mixing catches up. Tail-heaviness alone does not predict crossover horizon — what matters is the ratio of long-run regime-mixing volatility to single-regime σ.

The notebook §7 markdown was rewritten **after** seeing the numbers (per the same evidence-led discipline as RL-035).
