# RL-006 — Historical Risk Analytics

## Objective

Implement core risk metrics as pure functions operating on `pd.Series` inputs.

## Scope

**Included:**
- `src/analytics/risk_metrics.py` — all metric functions
- `src/analytics/__init__.py` — package init

**Excluded:**
- Visualization, Streamlit integration, Monte Carlo
- No DataFrame-level API — all functions take Series (`close` or `returns`)

## Functions

| Function | Input | Output | Purpose |
|---|---|---|---|
| `annualized_volatility` | returns Series | float | Annualized std of daily returns |
| `rolling_volatility` | returns Series, window | Series | Time-varying risk (default 21d) |
| `max_drawdown` | close Series | float | Worst peak-to-trough loss |
| `drawdown_series` | close Series | Series | Full drawdown curve |
| `sharpe_ratio` | returns Series, rfr | float | Risk-adjusted return |
| `return_statistics` | returns Series | dict | Distribution summary (mean, std, skew, kurtosis) |

## Decisions

- **Series-in, scalar/Series-out API**: Decouples metrics from dataframe column names. Any caller passes the right Series — no implicit column assumptions inside the module.
- **252 trading days**: Standard annualization convention for equities. Works for crypto too (approximation is acceptable for MVP).
- **scipy.stats for skew/kurtosis**: More reliable than pandas built-ins for these specific statistics.
- **No custom classes**: Plain functions. Avoids abstraction overhead for what is fundamentally a set of stateless calculations.

## Acceptance Criteria

- All functions return correct types (float, Series, dict).
- Functions work with the output of `process.py`'s `add_returns()`.
- No side effects, no storage, no state.
