# RL-009 — Test Suite

## Objective

Automated tests for the core analytics modules and data pipeline to prevent regressions.

## Scope

**Included:**
- `tests/test_risk_metrics.py` — 17 tests for all risk metric functions
- `tests/test_monte_carlo.py` — 12 tests for simulation, VaR/CVaR, scenarios
- `tests/test_data_pipeline.py` — 11 tests for processing chain and validation

**Excluded:**
- Streamlit UI testing
- Network-dependent tests (yfinance mocked)
- Storage tests (already in RL-004's `test_data_storage.py`)

## Test Summary

| Module | Tests | Coverage |
|---|---|---|
| `test_risk_metrics.py` | 20 | exact values, edge cases, empty series, zero-vol, monotonic price |
| `test_monte_carlo.py` | 18 | exact VaR/CVaR, prob consistency, shape, seed reproducibility, invalid inputs |
| `test_data_pipeline.py` | 15 | column normalization, sorting, NaN handling, returns correctness, index monotonicity |
| `test_data_storage.py` | 13 | (existing from RL-004) |
| **Total** | **75** | |

## Decisions

- **Deterministic fixtures via np.random.seed**: no flaky tests from random data.
- **Mocked yfinance in validation tests**: tests run offline, no API dependency.
- **Floating-point tolerance for zero-vol edge case**: `< 1e-10` instead of `== 0.0` due to IEEE 754 precision.
- **Exact value assertions**: seeded fixtures produce deterministic outputs — tests assert against computed values, not just types/signs.
- **prob_target/prob_gain consistency fix**: both now use strict `>` inequality. Verified by exact equality test.
- **sortino_ratio bug fix**: empty downside returns now returns 0.0 instead of NaN.
- **No Streamlit tests**: testing widget rendering adds complexity with no MVP value. Manual verification is sufficient.

## Acceptance Criteria

- `pytest tests/ -v` passes with 0 failures.
- All tests run without network access (mocked where needed).
- Tests cover contracts, edge cases, and semantic correctness.
