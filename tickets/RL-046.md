# RL-046 — EVT Diagnostic Suite

## Status: Done

## Problem
The EVT implementation (`evt.py`) fits a GPD via `scipy.stats.genpareto.fit` and reports point estimates of VaR/CVaR with zero validation. The 0.95 threshold is hardcoded with no justification. No goodness-of-fit check, no uncertainty quantification, no treatment of the iid violation from volatility-clustered extremes. This is the difference between calling an API and doing EVT.

## Solution
Six pure diagnostic functions added to `src/analytics/evt.py`. Each is stateless, takes returns (or GPD params) in and returns data out — no plotting, no side effects. Notebooks plot and interpret.

## Functions added

### `mean_residual_life(returns, n_thresholds=50) → DataFrame`
Computes E[X - u | X > u] across thresholds with 95% CI. Linearity above u₀ supports the GPD assumption. Used to justify threshold selection visually.

### `gpd_stability(returns, quantile_range) → DataFrame`
Fits GPD at 30 thresholds (0.80–0.98) and reports ξ, σ, σ* = σ - ξu, and n_exceedances. Both ξ and σ* should stabilize above the correct threshold. The existing 0.95 choice is marked for comparison.

### `gpd_qq(gpd_params, returns) → (theoretical, empirical)`
QQ plot data: theoretical GPD quantiles vs sorted empirical exceedances. Points off the diagonal at the upper end indicate the GPD underestimates extreme losses.

### `gpd_ks_test(gpd_params, returns) → dict`
Kolmogorov-Smirnov test of exceedances against the fitted GPD CDF. Returns statistic, p-value, and pass/fail at 5%. Formal goodness-of-fit.

### `gpd_bootstrap_ci(returns, threshold_quantile, n_boot=1000, ci=0.95) → dict`
Resamples exceedances with replacement, refits GPD per sample, reports 95% CI on shape and scale. Critical for honest reporting: "ξ = 0.15 [0.03, 0.31]" vs bare "ξ = 0.15".

### `decluster_pot(returns, threshold_quantile, run_length=10) → dict`
Runs declustering: groups consecutive exceedances into clusters (separated by ≥ run_length non-exceedances), keeps cluster maxima. Addresses the iid violation from volatility clustering. Returns declustered exceedances, cluster count, and raw count for comparison.

## Tests
44/44 EVT tests pass (22 existing + 22 new).

Key test strategies:
- MRL: output shape, positivity, CI containment, threshold monotonicity
- Stability: output shape, n_exceedances decreasing, custom quantile range respected
- QQ: equal-length arrays, both sorted
- KS: expected keys, passes on data drawn from a known GPD, p-value in [0, 1]
- Bootstrap: CI keys + structure, CI contains point estimate (within tolerance), scale CI positive, raises on insufficient data
- Declustering: expected keys, fewer clusters than raw exceedances, positive exceedances, longer run_length → fewer clusters

## Decisions
- **Pure functions, no plots.** Plotting is a notebook concern. Functions return data; notebooks call matplotlib. Keeps `evt.py` testable and dependency-light.
- **MRL over Hill plot.** MRL is more interpretable and directly tests the GPD linearity assumption. Hill plot only estimates ξ, not the full GPD validity.
- **KS test, not Anderson-Darling.** KS is available in scipy for arbitrary distributions via `kstest`. AD would need a custom implementation for GPD. KS is sufficient as a first-pass formal check.
- **Bootstrap CI, not profile likelihood CI.** Bootstrap is simpler, assumption-free, and works for both shape and scale simultaneously. Profile likelihood is more efficient but harder to implement correctly for GPD.
- **Declustering as optional, not default.** `fit_gpd` still works on raw exceedances by default. `decluster_pot` returns filtered exceedances that can be passed to `genpareto.fit` manually. This avoids breaking the existing API or silently changing MS-GARCH's per-regime GPD behavior.
