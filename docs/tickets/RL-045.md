# RL-045 — Redefine `data/` as a Flat yfinance Cache + Fix CI Build Backend

## Status: Done

## Goals
1. Make the `data/` directory **actually do something**. Before: `data/{raw,interim,processed}/` existed as scaffolding but nothing persisted to disk during normal runs. The app was pure in-memory, `src/data/storage.py` only wrote to `processed/` and was only invoked by `main.py`.
2. Unblock CI. `pyproject.toml`'s build backend was `setuptools.backends._legacy:_Backend` (module does not exist) — `pip install -e .` has failed on every CI run for weeks, so no CI test run has actually executed.

## Design
**Single flat `data/` directory, gitignored, used as a yfinance cache only.**

- One file per `(ticker, start_date, end_date)` tuple: `data/{TICKER}_{start}_{end}.csv`.
- No `raw/` vs `processed/` split — processing (clean + returns) is cheap and deterministic, not worth caching.
- Cache is a local-dev accelerator. Streamlit Cloud's filesystem is ephemeral, so caching there is a no-op by design. Documented in `src/data/storage.py` docstring.

## Changes

### `src/config.py`
- Replaced `DATA_RAW_DIR` and `DATA_PROCESSED_DIR` with a single `DATA_DIR = PROJECT_ROOT / "data"`.

### `src/data/storage.py` (rewritten)
Public API:
- `get_or_fetch(ticker, start_date=None, end_date=None) -> DataFrame` — primary entry point. Cache hit returns the DataFrame; miss calls `fetch_asset_data` and writes the CSV.
- `clear_cache() -> int` — delete all cached files.
- `list_cached() -> list[str]` — inspect the cache.

Removed (unused / policy-bound): `save_asset_data`, `load_asset_data`, `has_cached_data`, `delete_asset_data`, `list_cached_assets`, `cleanup_all_cached_data`, `get_cache_size_bytes`. The old "no-accumulation overwrite-in-place" policy is gone — the cache can grow across different date ranges; user clears manually via `main.py --purge` or `clear_cache()`.

Default-date normalization lives in `_resolve_dates()` inside `storage.py` (mirrors `fetch.py`'s logic), ensuring "5 years ago → today" keys stably within a single day.

### `src/data/fetch.py`
No change — remains pure (ticker + resolved dates → DataFrame, no side effects).

### `app/streamlit_app.py`
- Swapped `fetch_asset_data(...)` → `get_or_fetch(...)`. Transparent: same signature, adds caching on local runs, no-op on Cloud.

### `main.py`
- Uses `get_or_fetch` + `clear_cache`. Removed the pre-fetch "existing data will be overwritten" message (no longer applicable — different date ranges now coexist).

### `tests/test_data_storage.py` (rewritten)
- `TestGetOrFetch`: miss calls fetch + caches; hit skips fetch; different date ranges produce different cache keys.
- `TestCacheAdmin`: `clear_cache` / `list_cached` behavior.
- Mocks `fetch_asset_data` so tests never hit yfinance. Uses `monkeypatch` on `storage.DATA_DIR` pointed at `tmp_path`.

### `.gitignore`
- Replaced `data/raw/`, `data/interim/`, `data/processed/` with a single `data/`.

### `pyproject.toml` (CI fix)
- Changed `build-backend = "setuptools.backends._legacy:_Backend"` → `"setuptools.build_meta"`. The former is not a real module; `pip install -e .` raised `BackendUnavailable: Cannot import 'setuptools.backends._legacy'` on every CI run since it was introduced.

### Filesystem cleanup
- Removed `data/raw/`, `data/interim/`, `data/processed/` (empty or single stale file). Directory `data/` is recreated on demand by `storage.DATA_DIR.mkdir()`.

## Decisions
- **Flat directory, not `raw/processed/` split.** The split implies a pipeline that doesn't exist. Processing is always in-memory.
- **CSV not Parquet.** A 5-year daily OHLCV is ~80KB. Parquet would need `pyarrow` as a new dependency for cosmetic improvement.
- **Gitignored (not committed).** User preference. Cache is a local-dev accelerator, not a deliverable artifact. Data is reproducible from yfinance; committing would add churn + size with no audit-trail benefit since yfinance is the source of truth.
- **Cache admin exposed, not hidden.** `clear_cache` / `list_cached` + `main.py --purge` give users control without adding UI complexity.
- **Bundle CI fix with the refactor.** The build-backend typo is a one-line fix touching the same infra; splitting into its own ticket would delay the first working CI run needlessly.
