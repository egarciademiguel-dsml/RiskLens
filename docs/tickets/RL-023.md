# RL-023 — EVT tail risk + XGB model removal

## Status: Done

## Goal
Add Extreme Value Theory (GPD/POT) for theoretically justified tail risk estimation. Remove the broken XGBoost realized vol model which was functionally identical to constant vol.

## What changed

### EVT module (new)
- **`src/analytics/evt.py`**: Peaks Over Threshold method — fits Generalized Pareto Distribution to tail losses. Provides `evt_var()` and `evt_cvar()` with proper POT formulas. Includes `normal_var()`/`normal_cvar()` baselines for comparison. `evt_summary()` wraps everything with tail type interpretation.

### XGB model removal
- **Deleted `src/analytics/vol_rvol.py`**: The model predicted one flat sigma via XGBoost, applied it to all timesteps — functionally identical to constant vol with a fancier sigma estimate. No time-varying dynamics, no per-step updates, no value over existing models.
- **`src/analytics/monte_carlo.py`**: Removed `rvol` from `_VOLATILITY_MODELS` registry, removed rvol imports and n_days cap logic.
- **`src/analytics/backtesting.py`**: Removed `rvol_fit()` and `fit_rvol` import.
- **`app/streamlit_app.py`**: Removed XGBoost option from volatility model selector, all rvol-related UI code and imports.
- **Deleted `tests/test_vol_rvol.py`**: All rvol tests removed.
- **`tests/test_evt.py`**: 22 new tests covering GPD fitting, EVT VaR/CVaR, Normal baselines, comparison, and summary.

## Decisions
- XGB as a volatility model was "ML for the sake of ML" — it added complexity without dynamic behavior. Will be replaced in RL-024 with conditional quantile regression (a defensible, additive use of ML for direct VaR estimation).
- EVT uses GPD with POT method (standard in quantitative risk management). Default threshold at 95th percentile of losses.
- EVT sits alongside MC pipeline as an independent tail estimator, not inside it.
