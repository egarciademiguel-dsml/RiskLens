# RL-034 — MS-GARCH Unified: Global GARCH + Per-Regime Omega + Per-Regime GPD

## Status: Done

## Goal
Fix the structural alpha=0 problem in BTC crisis regime by fitting GARCH(1,1) **once** on the full return series and using regime labels only to (a) reparametrize the unconditional level via per-regime omega and (b) discretize the tail via per-regime GPD on the global standardized residuals.

## Scope
- Refactor `fit_ms_garch` only — `generate_log_returns` is unchanged because the per-regime parameter contract is preserved.
- Document the shared-persistence assumption (alpha+beta identical across regimes) with theoretical and empirical justification.
- Backtest BTC and save standalone results report.

## Files changed
- `src/analytics/ms_garch.py` — `fit_ms_garch` rewritten; helpers `_compute_standardized_residuals`, `_constant_garch_fallback` retained but no longer called from main path.
- `docs/decisions/ms_garch_unified.md` (new) — assumption justification.
- `reports/ms_garch_unified_results.md` (new) — backtest comparison.
- `scripts/run_ms_garch_unified_backtest.py` (new) — reproducer.

## Acceptance criteria
- Global GARCH fits on the pooled series with sensible (alpha, beta).
- Per-regime omega derived via variance targeting: `omega_k = sigma²_k * (1 - persistence)`.
- Per-regime GPD fitted on global standardized residuals filtered by regime.
- BTC backtest improves on RL-033 results.
- Decision documented with literature references.

## Decisions
- Shared persistence across regimes: theoretically sound (Hamilton & Susmel 1994 SWARCH; Engle & Mezrich 1996 variance targeting; eliminates Gray 1996 path-dependence) and empirically supported (Haas/Mittnik/Paolella 2004; Klaassen 2002; Marcucci 2005; Ardia et al. 2018 MSGARCH package). See `docs/decisions/ms_garch_unified.md`.
- Variance targeting at the regime level is the cleanest way to inject regime structure while keeping volatility dynamics globally identifiable.
- GPD fitted on *global* standardized residuals filtered by regime (not on regime-local residuals) so the standardization is consistent with the simulation engine.
