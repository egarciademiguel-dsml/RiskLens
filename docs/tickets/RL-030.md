# RL-030 — Fix `conservative_rank` Bug in Backtest Comparison

## Status: Done

## Goal
The model comparison ranking in `compare_models()` was inverted: the model with the **highest** breach rate was labeled most risk-conservative. Restore correct semantics so the lowest breach rate (closest to confidence target without overshooting) wins.

## Scope
- Fix the rank lambda only.
- Do not touch the calibration ranking, the breach test logic, or the backtest engine.

## Files changed
- `src/analytics/backtesting.py` — removed inverted sign in the conservative-rank lambda.

## Acceptance criteria
- Conservative rank #1 goes to the model with the lowest breach rate (most conservative VaR).
- Calibration rank still rewards models closest to the confidence target.
- Notebook comparison table re-renders without manual reinterpretation.

## Decisions
- "Most conservative" = predicts deepest losses = breaches least often. Inverting that was a copy-paste sign error.
