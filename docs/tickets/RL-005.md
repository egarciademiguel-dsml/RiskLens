# RL-005 — Documentation Cleanup

## Objective

Simplify project documentation to be professional, maintainable, and free of process bloat.

## Scope

**Included:**
- Rewrite `docs/workflow.md` — removed role formalism, simplified to practical workflow
- Rewrite `docs/project_overview.md` — tightened, removed ticket cross-references
- Simplify `docs/ticket_registry.md` — dropped Order column, sequential listing
- Clean up `docs/tickets/RL-001.md` — removed Extension/Prompt Tracking appendages
- Delete `docs/prompts_registry.md` — replaced by decision tracking in tickets

**Excluded:**
- Code changes
- Ticket content for RL-002, RL-003, RL-004 (already clean enough)

## Decisions

- **Removed prompts registry**: 634 lines of verbatim LLM prompts adds no value to the repo. Traceability comes from decision records in each ticket, not raw prompt storage.
- **Dropped role formalism**: Stakeholder/Architect/Implementer roles are meaningless in a solo project. Kept the workflow steps themselves.
- **Dropped ticket reorder section**: Confusing to explain RL-003 runs before RL-002. Just list tickets sequentially — the registry is a log, not an execution plan.
- **Added Decisions section to ticket template**: Each ticket now records key choices and why. This is what reviewers and recruiters actually care about.

## Acceptance Criteria

- `docs/prompts_registry.md` is deleted.
- All docs are concise, professional, and free of placeholders.
- Ticket registry is clean and sequential.
- Workflow doc is practical, not ceremonial.
