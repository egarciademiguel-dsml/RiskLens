# RL-010 — CI + Deployment

## Objective

Automated test pipeline and Streamlit Cloud deployment config, ready to activate when the GitHub repo is created.

## Scope

**Included:**
- `.github/workflows/tests.yml` — pytest on push/PR to main/master
- `.streamlit/config.toml` — dark theme, headless server config
- README deployment instructions (added in RL-012)

**Excluded:**
- Actual GitHub repo creation (user handles manually)
- Streamlit Cloud linking (requires live repo)
- CI badge (added in RL-012 README rewrite)

## Decisions

- **Python 3.11 in CI**: matches common Streamlit Cloud runtime; broad compatibility.
- **No caching in CI**: pip install is fast enough for 19 deps; avoids cache invalidation complexity.
- **Dark theme default**: matches financial dashboard conventions; looks more professional.
- **Headless server**: required for Streamlit Cloud deployment.

## Acceptance Criteria

- Workflow file exists and is valid YAML.
- Streamlit config applies dark theme.
- Both are ready to work on push to GitHub with zero modifications.
