# RL-026 ŌĆö Refactor Shock Generation: Model-Owned Innovations

## Status: Done

## Goal
All 4 MC volatility models produced nearly identical results because they shared the same shock distribution (Normal or Student-t). Each model only differed in how it scaled sigma[t], but the tail shape was identical. This ticket decouples innovation distributions from the dispatcher so each model owns its shock generation internally.

## What changed

### Signature change (all models)
**Before:** `generate_log_returns(shocks, returns, seed, **kwargs)`
**After:** `generate_log_returns(n_days, n_simulations, returns, seed, **kwargs)`

Each model now generates its own shocks internally instead of receiving pre-generated shocks from the dispatcher.

### `src/analytics/monte_carlo.py`
- Removed `distribution` and `df_t` parameters from `simulate_paths()`
- Removed shock generation block ŌĆö dispatcher now passes (n_days, n_simulations) to models
- `fit_t_distribution()` kept as utility for display

### `src/analytics/vol_constant.py`
- Generates Normal(0,1) shocks internally ŌĆö the baseline Gaussian assumption

### `src/analytics/vol_garch.py`
- Generates Student-t shocks internally (auto-fitted df from returns)
- Added `innovation` param: `"t"` (default) or `"normal"` for controlled comparisons
- Added `df_t` param for explicit degrees of freedom override

### `src/analytics/regime_hmm.py` / `regime_gmm.py`
- Generate Normal(0,1) shocks internally ŌĆö regime detection provides differentiation, not tail shape

### `app/streamlit_app.py`
- Removed `dist_choice` radio button (Normal/Student-t toggle)
- GARCH info box now shows Student-t df alongside GARCH parameters
- Model labels updated to reflect owned distributions

### Tests
- `tests/test_monte_carlo.py`: Removed `TestStudentTSimulation` class and `test_invalid_distribution_raises`. Added `test_garch_heavier_tails_than_constant` (kurtosis test) and `test_garch_normal_innovation`. Simplified `TestCrossModelProperties` (no distribution axis).
- `tests/test_regime_hmm.py`: Removed `test_hmm_with_t_shocks`
- `tests/test_regime_gmm.py`: Removed `test_gmm_with_t_shocks`

## Decisions
- Constant vol = Normal shocks (the null model, always Gaussian)
- GARCH = Student-t shocks (GARCH standardized residuals are known to be heavy-tailed)
- HMM/GMM = Normal shocks (their value is regime detection, not tail modeling)
- The `innovation` kwarg on GARCH enables controlled comparisons in the notebook (RL-028)
