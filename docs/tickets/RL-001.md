# RL-001 — Project Bootstrap

## Objective

Create the initial project structure, configuration files, and foundational documentation.

## Scope

**Included:**
- Directory structure (`src/`, `tests/`, `notebooks/`, `docs/`, `app/`)
- Configuration files (`pyproject.toml`, `requirements.txt`, `.gitignore`, `.pre-commit-config.yaml`)
- Minimal Streamlit entry point (`app/streamlit_app.py`)
- Documentation (`README.md`, `docs/workflow.md`, `docs/project_overview.md`)

**Excluded:**
- Business logic, data processing, analytics code

## Files Created

- `app/streamlit_app.py`
- `requirements.txt`
- `pyproject.toml`
- `.pre-commit-config.yaml`
- `.gitignore`
- `README.md`
- `docs/workflow.md`
- `docs/project_overview.md`
- `docs/tickets/RL-001.md`

## Acceptance Criteria

- All directories and files exist with correct content.
- `streamlit_app.py` displays title and bootstrap message.
- Documentation is complete.

## Decisions

- **Ruff over Black/isort**: single tool for linting + formatting, less config overhead.
- **Streamlit**: lightweight, Python-native UI — ideal for data science MVP.
- **Flat src/ layout**: simple enough for MVP, avoids premature packaging.
