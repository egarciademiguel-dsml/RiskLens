# RL-019 — Realized Volatility Regression (XGBoost)

## Objective

Predict forward-looking realized volatility using XGBoost on engineered rolling features. Three horizons: 5d, 10d, 21d. Integrates into `simulate_paths()` as a volatility source.

## Scope

**Volatility engine (`src/analytics/vol_rvol.py`):**
- `engineer_vol_features(returns)` → DataFrame with rolling features (vol/mean/skew/kurtosis at multiple windows)
- `build_target(returns, horizon)` → forward realized vol series
- `fit_rvol(returns, horizon)` → fitted model dict (XGBoost model, scaler, feature columns, train metrics)
- `predict_current_vol(rvol_result, recent_returns)` → predicted forward vol (scalar)
- `generate_log_returns(shocks, returns, seed, rvol_params, **kwargs)` → log returns using predicted vol + global drift

**MC integration:**
- New `volatility_model="rvol"` in registry
- Uses global mu for drift, ML-predicted sigma for volatility

**Streamlit:**
- UI toggle for "ML Volatility (XGBoost)"
- Horizon selector (5d, 10d, 21d)
- Display predicted vol and model R²

**Tests:**
- Feature engineering shape and no-NaN
- Model fits without error for all 3 horizons
- Predicted vol is positive
- MC integration produces valid paths
- R² > 0 (better than mean baseline)

- Predicted volatility is positive and reasonable
