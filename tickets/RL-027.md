# RL-027 — MS-GARCH: Markov-Switching GARCH with EVT Tails

## Status: Done

## Goal
New top-tier model combining HMM regime detection + per-regime GARCH(1,1) + per-regime EVT/GPD tails. Each component adds something the previous structurally can't capture: HMM identifies market states, GARCH models within-state volatility dynamics, EVT shapes the tail distribution per state.

## What changed

### New module — `src/analytics/ms_garch.py`
- `fit_ms_garch(returns, n_regimes, seed, threshold_quantile, min_regime_obs)`: fits HMM, then per-regime GARCH(1,1) on regime-filtered returns, then GPD on GARCH standardized residuals per regime. Graceful fallbacks for small regimes.
- `generate_evt_shocks(gpd_params, size, rng)`: semi-parametric CDF inversion — Normal bulk + GPD left tail. GPD fitted in standardized-residual space so no rescaling needed.
- `generate_log_returns(n_days, n_simulations, returns, seed, ms_garch_params)`: per-timestep loop with regime-specific GARCH variance update, EVT/Normal shock generation, and HMM regime transitions. Variance state carries across regime transitions.
- `_constant_garch_fallback(sigma)`: degenerate GARCH (alpha=0, beta=0) for small regimes.
- `_compute_standardized_residuals(returns, garch_params, mu)`: forward GARCH pass to get conditional variances and standardize returns.

### Registry — `src/analytics/monte_carlo.py`
- Added `"ms_garch"` to `_VOLATILITY_MODELS` registry.
- Imported `fit_ms_garch` for re-export.

### Backtesting — `src/analytics/backtesting.py`
- Added `ms_garch_fit(returns_window, seed, n_regimes)` fit function for rolling-window backtesting.

### Tests — `tests/test_ms_garch.py` (23 new tests)
- `TestFitMsGarch`: output keys, per-regime param counts, GARCH keys, current regime range, multi-regime (2, 3)
- `TestGenerateEvtShocks`: shape, no NaNs, heavier left tail than Normal, Normal-like bulk, thin tail shape
- `TestFallbacks`: fallback keys, degenerate GARCH values, forced fallback with high min_regime_obs
- `TestMsGarchIntegration`: simulate_paths shape, positive prices, reproducibility, differs from constant and GARCH, CVaR <= VaR, pre-fitted params
- `TestMsGarchBacktest`: ms_garch_fit produces valid kwargs

## Decisions
- Regime-filtered returns treated as contiguous for GARCH fitting (standard in MS-GARCH literature: Haas, Mittnik, Paolella 2004)
- min_regime_obs=100 default. Below threshold: GARCH degenerates to constant vol. Below 50 obs: GPD skipped (Normal innovations)
- Variance state carries across regime transitions for physical continuity (no variance reset on regime switch)
- Variance clamped at 10x regime long-run variance to prevent explosion
- GPD fitted on standardized residuals (not raw returns) so shock scale matches sigma[t] * shock[t] framework
