# RL-028 — Controlled Comparison Notebook

## Status: Done

## Goal
Rebuild model_deep_dive.ipynb as a controlled experiment isolating tail effect vs vol effect vs combined, across 3 assets. This is the analytical backbone that justifies the 3-tier model selection.

## What changed

### `notebooks/model_deep_dive.ipynb` (complete rewrite)

**Structure (28 cells, 8 sections):**

1. **Data & Setup**: 3 assets (BTC-USD, SPY, NVDA), 5yr data each. Reports trading days, ann. vol, kurtosis per asset.

2. **Comparison 1 — Pure tail effect**: Fix GARCH, vary innovation (Normal vs Student-t). Isolates fat-tail impact. Table + KDE overlay + left-tail survival (log scale) per asset.

3. **Comparison 2 — Pure vol effect**: Fix Normal shocks, vary vol model (Constant vs GARCH). Isolates volatility dynamics impact. Table + KDE overlay per asset.

4. **Comparison 3 — 3 tiers head-to-head**: Baseline vs GARCH+Student-t vs MS-GARCH+EVT. Full metrics table (VaR, CVaR, P(loss>30%), P(loss>50%), kurtosis). VaR bar charts, KDE, left-tail survival, percentile fan charts per tier.

5. **MS-GARCH model details**: Per-regime GARCH params (alpha, beta, persistence), GPD shape/scale, transition matrices. Printed for all 3 assets.

6. **Tail risk audit**: EVT and XGBoost 1-day VaR comparison across 3 assets. Normal vs EVT vs XGB table.

7. **Backtest validation**: compare_models() on BTC-USD with the 3 tiers + EVT backtest. Kupiec and Christoffersen results.

8. **Conclusions**: What each layer adds, why the old approach failed, the 3-tier narrative.

## Decisions
- 3 assets chosen for different tail/vol profiles: BTC (extreme), SPY (moderate), NVDA (volatile equity)
- Controlled comparisons isolate one variable at a time — cleaner than full combinatorial grid
- GARCH `innovation="normal"` kwarg (added in RL-026) enables the controlled tail comparison
- Percentile fan charts shown per-tier side-by-side for BTC (most interesting tails)
- KDE plots trimmed at p99 cutoff for readability
