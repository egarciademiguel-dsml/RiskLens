# RL-015 — Test Refactor, Mathematical Reference & Model Comparison

## Objective

1. Replace brittle hardcoded test values with property-based assertions that hold across all model configurations.
2. Add a concise mathematical reference document covering all implemented models, metrics, and assumptions.
3. Add a notebook comparing the four model configurations side by side.

## Scope

**Test refactor:**
- Removed 4 exact-value tests (VaR, CVaR, prob_gain, prob_loss, mean_final_price)
- Kept 1 seeded regression test for the constant+normal code path
- Replaced with property-based tests: VaR < 0, CVaR ≤ VaR, prob in [0,1], min ≤ mean ≤ max
- Added `TestCrossModelProperties` class with parametrized tests across all 4 model combos (normal/t × constant/garch)
- Consolidated `TestInvalidInputs` (merged distribution + volatility_model validation)
- Total: 61 tests (was 46)

**Mathematical reference (`docs/mathematical_reference.md`):**
- Return types (simple vs log)
- All risk metrics with formulas
- GBM framework and Jensen's correction
- Shock distributions: Normal, Student-t (implemented), Skewed-t/GHD/EVT (future)
- Volatility models: Constant, GARCH(1,1) (implemented), EGARCH/GJR/HMM (future)
- Model combination matrix
- Assumptions table
- Interpretation guide

## Decisions

- **Kept one regression test**: `test_var_deterministic_regression` pins the constant+normal code path to catch accidental numerical changes. All other exact values removed.
- **Parametrized cross-model tests**: ensures core invariants (CVaR ≤ VaR, probs sum to 1, prices positive, scenarios sum to 1) hold for all 4 model combinations.
- **Math doc scope**: covers implemented + future models. Concise enough to scan in 5 minutes, detailed enough for a recruiter or interviewer to assess understanding.

**Model comparison notebook (`notebooks/model_comparison.ipynb`):**
- Runs all 4 combos (Normal/t × Constant/GARCH) on same asset and seed
- Summary table: VaR, CVaR, P(Gain), mean/median/min/max price
- VaR & CVaR bar chart comparison
- Overlaid final price KDE distributions
- 2×2 MC paths grid (200 sample paths each)
- 2×2 scenario bucket charts
- Tail risk deep dive: percentiles, kurtosis, skewness
- Key takeaways section
