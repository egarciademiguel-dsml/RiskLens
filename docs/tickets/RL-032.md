# RL-032 — Diagnose MS-GARCH Backtest Failure

## Status: Done

## Goal
Explain *why* MS-GARCH+EVT produced the deepest 252-day VaR yet failed the daily backtest with 11.7% breach rate. Pure diagnostic ticket — no code fix.

## Scope
- Notebook section that inspects per-regime GARCH parameters, breach concentration by regime, and mean predicted VaR by tier.
- Standalone results report.
- No production code changes.

## Files changed
- `notebooks/model_deep_dive.ipynb` (section 7b: MS-GARCH Diagnosis)
- `reports/ms_garch_diagnosis_results.md` (new)

## Acceptance criteria
- Per-regime GARCH quality table for all three assets.
- Breach concentration split by regime label for BTC.
- Daily mean predicted VaR comparison across tiers.
- Root cause identified.

## Decisions / Findings
- Root cause: per-regime GARCH fit on regime-filtered slices destroys serial dependence and collapses ARCH coefficient to 0 in sparse regimes (BTC crisis: 44 obs, alpha=0.0000).
- BTC breach concentration: Calm 3.3%, Crisis 34.1% — failure is concentrated where the model is supposed to be most reactive.
- NVDA both regimes degenerate to constant-vol fallbacks.
- The diagnosis explicitly motivated RL-033 (transition-first fix) and RL-034 (unified GARCH).
