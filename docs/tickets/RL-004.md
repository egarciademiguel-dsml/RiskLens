# RL-004 — No Persistent Data Accumulation Strategy

## ID

RL-004

## Title

No Persistent Data Accumulation Strategy

## Objective

Ensure RiskLens does not accumulate unnecessary local market data over time. The app uses only the minimum data needed for current analysis, avoids building an ever-growing local store, and keeps data handling lightweight, controlled, and inspectable.

## Rationale

RiskLens is an analysis app, not a data warehouse. Without an explicit retention policy, repeated runs of `main.py` or the Streamlit app would silently grow `data/processed/` with no upper bound. This ticket makes the no-accumulation intent explicit in both code and documentation.

## Chosen Retention Strategy

**Overwrite-on-refresh, one file per asset maximum.**

- Each asset gets exactly one CSV file (`{TICKER}_daily.csv`).
- Re-fetching the same asset overwrites the existing file.
- A `--purge` CLI flag removes all cached files.
- Storage utilities expose inspection (list, size) and cleanup operations.
- No raw data is persisted; only processed output is cached.

## Storage Behavior

| Action | Result |
|---|---|
| First fetch for a ticker | Creates `data/processed/{TICKER}_daily.csv` |
| Re-fetch same ticker | Overwrites the existing file |
| `--purge` flag | Deletes all `*_daily.csv` in `data/processed/` |
| No fetch | No file created |
