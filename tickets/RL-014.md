# RL-014 — GARCH(1,1) Volatility Model

## Objective

Replace constant-sigma assumption in Monte Carlo with GARCH(1,1) time-varying volatility. Real markets exhibit volatility clustering (high-vol follows high-vol). Combined with Student-t shocks from RL-013, this gives GARCH + Student-t — a standard quant finance simulation model.

## Scope

**Included:**
- `simulate_paths()` accepts `volatility_model="garch"` and optional `garch_params`
- `fit_garch(returns)` helper: fits GARCH(1,1) via `arch` library, returns omega/alpha/beta/persistence/long-run vol
- Day-by-day variance simulation: `sigma²(t) = omega + alpha * resid²(t-1) + beta * sigma²(t-1)`
- Jensen's drift correction uses per-day sigma_t: `drift_t = mu - 0.5 * sigma_t²`
- Streamlit sidebar radio: Constant vs GARCH(1,1)
- Info banner with fitted GARCH params
- 11 new tests (7 simulation + 4 fit helper)
- `arch` added to requirements.txt

**Excluded:**
- No EGARCH, GJR-GARCH, or other GARCH variants
- No test refactor (separate ticket RL-015)

## Decisions

- **arch library for fitting**: standard Python GARCH library. MLE fitting with `disp="off"` to suppress optimizer output.
- **Percentage returns for arch, decimal for simulation**: arch expects percentage-scale returns (e.g., 2.0 for 2%). Fitted params are converted back to decimal scale for simulation consistency.
- **Day-by-day loop for variance path**: each day's variance depends on the previous day's realized shock, so this cannot be vectorized. Performance is fine for 504 x 50000 (the max UI config).
- **Backward compatible**: `volatility_model="constant"` is the default, preserving all existing behavior and test values.
