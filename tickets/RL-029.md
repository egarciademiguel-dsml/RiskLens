# RL-029 — App Simplification to 3 Tiers

## Status: Done

## Goal
Replace the 4-model radio + distribution toggle + regime slider UI with a single 3-tier risk model selector: Baseline / GARCH+Student-t / MS-GARCH+EVT.

## What changed

### `app/streamlit_app.py` (major rewrite)

**Sidebar:**
- Removed `vol_choice` radio (Constant/GARCH/HMM/GMM), `dist_choice` radio (Normal/Student-t), `n_regimes` slider
- Replaced with single `tier` radio: Baseline (Constant + Normal) / GARCH + Student-t / MS-GARCH + EVT (Full Model)
- Added tier explanation expander describing what each tier adds

**Tier mapping:**
| Tier | volatility_model | model_kwargs |
|------|-----------------|--------------|
| Baseline | "constant" | {} |
| GARCH+t | "garch" | {"garch_params": fit_garch(returns)} |
| MS-GARCH+EVT | "ms_garch" | {"ms_garch_params": fit_ms_garch(returns)} |

**Info boxes:**
- Baseline: annualized vol
- GARCH: persistence, long-run vol, Student-t df
- MS-GARCH: per-regime GARCH params (alpha, beta, persistence), GPD shape, current regime

**Regime overlay:**
- MS-GARCH tier shows HMM regime labels as colored background on the price chart

**Backtest tab:**
- Maps tier to fit function: constant_fit / garch_fit / ms_garch_fit
- Removed HMM/GMM standalone backtest options

**Removed:**
- All HMM/GMM standalone imports and UI code
- Distribution toggle (each model now owns its innovation distribution)
- Regime count slider (MS-GARCH uses n_regimes=2 by default)

**Kept unchanged:**
- EVT tail risk tab (audit tool)
- XGBoost VaR tab (audit tool)
- All risk metrics, drawdown, Monte Carlo visualization logic

## Decisions
- 3 tiers chosen to match the analytical narrative from RL-028 comparison notebook
- MS-GARCH defaults to n_regimes=2 (most robust across assets)
- HMM/GMM standalone options removed — their value is captured inside MS-GARCH
- EVT and XGB tabs kept as independent audit tools per ML philosophy
