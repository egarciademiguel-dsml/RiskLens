# RL-002 — Core Data Ingestion Layer

## ID

RL-002

## Title

Core Data Ingestion Layer

## Objective

Implement the data ingestion pipeline: ticker validation, OHLCV fetching via yfinance, data cleaning, returns computation, and CSV export.

## Includes

- `src/data/validate.py` — ticker validation against yfinance.
- `src/data/fetch.py` — daily OHLCV download with configurable date range.
- `src/data/process.py` — cleaning, standardization, returns and log-returns.
- `src/config.py` — project paths and defaults.
- `src/utils.py` — directory setup utility.
- `main.py` — CLI entry point (`python main.py --ticker BTC-USD`).
- `data/raw/` and `data/processed/` directories.
- `src/__init__.py` and `src/data/__init__.py` for package resolution.

## Excludes

- Analytics, simulation, or modeling logic.
- Streamlit integration.
- Additional dependencies beyond `requirements.txt`.
- Any modification to RL-001 files not listed in documentation updates.

## Acceptance Criteria

- `python main.py --ticker BTC-USD` downloads data, processes it, saves CSV to `data/processed/`, and prints a preview.
- Invalid tickers are caught and reported.
- Output dataframe contains: date, open, high, low, close, volume, returns, log_returns.
- No missing values in the output.
- All prompts recorded in `docs/prompts_registry.md`.

## Notes

- yfinance may return multi-level columns depending on version; `fetch.py` handles both cases.
- Validation makes a lightweight 5-day request to confirm ticker existence.
