# RL-025 — XGBoost Temporal CV + EVT Backtesting

## Status: Done

## Goal
Two validation gaps: XGBoost hyperparameters were hardcoded without cross-validation, and EVT had no out-of-sample backtest. This ticket adds TimeSeriesSplit CV for XGBoost tuning and expanding-window walk-forward backtesting for EVT VaR.

## What changed

### XGBoost tuning — `src/analytics/xgb_var.py`
- `pinball_loss()`: quantile loss scoring function for CV evaluation.
- `tune_hyperparameters()`: grid search over max_depth, learning_rate, n_estimators using `TimeSeriesSplit(n_splits=3)`. Returns best params by pinball loss.
- `fit_quantile_model(tune=True)`: optional flag to run CV before fitting. Adds `best_params` to output dict. Default `tune=False` preserves backtest loop speed.

### EVT backtest — `src/analytics/backtesting.py`
- `backtest_evt_var()`: expanding-window walk-forward backtest for EVT/GPD VaR. Same pattern as `backtest_var()` and `backtest_quantile_var()`. Output compatible with `backtest_summary()`, Kupiec, and Christoffersen tests.

### Streamlit — `app/streamlit_app.py`
- Tail Risk tab: XGBoost now runs with `tune=True`, displays selected hyperparameters.
- Backtest tab: EVT added as a separate backtest with its own button, metrics, and chart.

### Tests
- `tests/test_xgb_var.py`: 8 new tests — pinball loss, tune_hyperparameters (returns dict, params in grid), fit with tune flag (best_params present/absent, still predicts).
- `tests/test_backtesting.py`: 6 new tests — EVT backtest output schema, breach dtype, breach consistency, backtest_summary compatibility, VaR sign.

## Decisions
- Tuning runs only in Streamlit dashboard (not inside backtest loop) — tuning inside each backtest iteration would multiply runtime ~18x
- Small grid (18 combos x 3 folds = 54 fits) — practical for interactive use
- EVT backtest uses same expanding-window pattern as MC backtests for consistency
- Parametric models (GARCH, HMM, GMM) unchanged — few parameters, low overfitting risk
